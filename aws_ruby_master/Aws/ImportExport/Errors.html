      Overview

      When AWS Import/Export returns an error response, the Ruby SDK constructs and raises an error. These errors have the following ancestors:

      • Aws::ImportExport::Errors::ServiceError
      • Aws::Errors::ServiceError
      • RuntimeError

      You can rescue all ImportExport errors using ServiceError:

      # rescue all errors from AWS Import/Export
begin
  # do stuff
rescue Aws::ImportExport::Errors::ServiceError
  # ...
end

      Request Context

      ServiceError objects have a #context method that returns information about the request that generated the error, including:

      • #params - The request params
      • #operation_name - Name of the API operation invoked
      • #http_request
      • #http_response
      • etc ...

      See ::Seahorse::Client::RequestContext for more information.

      Error Classes

      You can rescue a specific error using the error code:

      begin
  # do stuff
rescue Aws::ImportExport::Errors::BucketPermissionException
  # ...
end
